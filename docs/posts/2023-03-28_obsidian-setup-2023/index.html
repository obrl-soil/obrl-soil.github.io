Background

I‚Äôve seen a few posts and videos lately1 about how various people manage the avalanche of technical literature coming their way2. There are many helpful tips available in those posts, but nothing quite fit my own situation and needs end-to-end. So: this post is about how I currently manage a natural resource science-heavy reference library and associated knowledge generation infrastructure.

To be clear, I don‚Äôt necessarily love this setup. Its the best I‚Äôve managed so far though, so hopefully writing about what works and what could be better will be inspirational.

Requirements

I am:

  • a translational research worker with high anxiety and a short attention span

I need/want:

  • A setup that is compatible with my work IT environment but not dependent on it, because that is safe
  • A setup that is relatively easy to upgrade and extend and generally futz around with, because that is fun.

Setup

I am currently using:

  • Reference manager: Zotero, with the following extensions (‚Äòadd-ons‚Äô):
    • Zotfile, for reference and source document management. Offers a one-click solution for moving a file into a designated spot and renaming it according to a given pattern.
    • Better Bibtex, which assigns unique IDs to Zotero references and automates their export to a *.bib file that can be picked up by other software that doesn‚Äôt have close integration with Zotero.
  • Notes manager: Obsidian, with the following extensions (‚Äòcommunity plugins‚Äô):
    • Citations: picks up the *.bib file that Better Bibtex puts down, and allows me to generate a markdown-format literature note for each reference with a matching name, some nice metadata, and a pre-formatted space all ready for me to add my notes.
    • Admonition: (optional extra) makes it easy to add relative dates by e.g. typing in, e.g. @today, which then becomes a link named after today‚Äôs date.
    • Projects: (optional extra) This plugin is very new, so maybe not super stable in the medium term, but I really like the ability to create table, kanban, gallery and calendar views of my notes. Its currently about as Notion or Anytype as you can get with markdown powering everything.
  • Storage: Office 365 OneDrive. Its whatever, I get it for free through work, just about any cloud storage environment could be dropped in its place.
  • Authoring: Sometimes Obsidian, sometimes Quarto in RStudio, sometimes even (gasp) MS Word. The important part is that only Word and RStudio have full Zotero integration; other apps have to interact with it via the aforementioned *.bib file.

Zotero

I‚Äôve installed Zotero the usual way on a Windows machine. I have then added Zotfile, and set up my preferences as follows:

  • General Settings > Location of Files: I have chosen ‚ÄòCustom Location‚Äô, and instructed it to place any new linked file attachments in a particular folder in my OneDrive. I will call it ‚ú®The Hoard‚ú® going forward. The only important thing about ‚ú®The Hoard‚ú® is that it lives inside an Obsidian Vault. A Vault is just another folder, with some extra bits inside.
  • Renaming Rules: This determines what the names of linked file attachments look like once they‚Äôre moved into ‚ú®The Hoard‚ú®. I use the template {%a}_{%y}_{%t} , which is first author surname, year, title, separated by underscores. I have also ticked ‚Äúchange to lower case‚Äù, ‚Äútruncate title after . or : or ?‚Äù3 , set ‚Äúmaximum length of title‚Äù to 160, and maximum number of authors to 1. As such most of my linked files start out looking like, e.g. ‚Äúdarwin_1859_on_the_origin_of_species.pdf‚Äù.
    • I can‚Äôt leave well enough alone, so I tend to do a bit more manual tweaking üôÑ. I add an ‚Äúitem type‚Äù slug like _PAPER_ or _THESIS_ , I remove filler words like ‚Äòand‚Äô, ‚Äòthe‚Äô and ‚Äònovel‚Äô from the title, and I change underscores in titles to dashes, after Jenny Bryan‚Äôs advice. They come out looking a bit like, e.g. ‚ÄúDarwin_1859_BOOK_origin-species-1e.pdf‚Äù. I do this because I want ‚ú®The Hoard‚ú® to be navigable on its own.
Tip

Maximum length of title becomes important if you use cloud storage. OneDrive‚Äôs maximum path length is 400 characters and some client software makes a fuss past 260 characters, so if your Hoard folder is nested deeply you have less to work with.

  • Advanced Settings: I expanded the ‚ÄòOnly work with the following filetypes‚Äô option to cover some extra attachment file types I often use, e.g. MS Office formats, common image formats, ebooks, markdown, html and zip files. If you don‚Äôt do this, automated move-and-rename won‚Äôt work properly.
Note

This option seems to mostly exist to protect against unintended consequences from enabling another setting in General Options - ‚Äòsource folder for attaching new files‚Äô grabs the most recently modified file in a nominated folder. This seems like more trouble than its worth, so if you actually use it, please explain‚Ä¶

Continuing, I have added Better Bibtex, and left all the defaults except:

  • Automatic export: ‚ÄúWhen Idle‚Äù, delayed for 60 seconds. This is a compromise; I work with a lot of grey literature and other Old Stuff that has to be manually entered into Zotero. The default setting was fast enough that I was running into freeze-ups and sync errors because I was still editing a reference while BB was trying to re-export my whole library. Waiting longer means I sometimes have to manually trigger an export to make a new reference available outside Zotero.
  • The plugin is instructed to export my library to a *.bib file in Better Bibtex format that sits in the same folder as ‚ú®The Hoard‚ú®.

Other Zotero add-ons

These aren‚Äôt important, but they are nice.

  • Zutilo - allows mass editing of reference item tags and attachment paths
  • Scite - scite.ai integration - citation records with context! I like what they‚Äôre doing. The service is still a bit incomplete in my main research fields, but improving all the time. One day I‚Äôll convince my work to pay for it‚Ä¶
  • Word integration - eh, why wouldn‚Äôt you?
  • Mdnotes - not actually using this at the moment, but if you have a lot of notes already in Zotero and you want to move them into Obsidian, it would be very helpful.

Obsidian

I have installed the software and set up a vault in my OneDrive called, imaginatively, Work_Stuff.

Important

You may have to pay for a commercial license to use Obsidian. I‚Äôm not currently as I don‚Äôt qualify according to the EULA (NZ Crown Research Institutes are a weird public/private monstrosity, but fundamentally I‚Äôm employed as a non-profit government researcher). You may or may not have to depending on your circumstances, look into it and do the right thing.

‚ú®The Hoard‚ú® lives in a subfolder of Work_Stuff, as mentioned.

I have then installed the Citations community plugin, with the following settings:

  • Citation database format: BibLatex
  • Citation database path: points at the *.bib file exported from Zotero by BB.
  • Literature Note folder: do what you want here. I started a specific Notes - sources folder inside the Vault (at the same level as ‚ú®The Hoard‚ú®). You can get more specific and organise by project, or just store your notes in a big pile in the vault‚Äôs root directory if you‚Äôre a filthy zoomer who doesn‚Äôt understand file systems, I‚Äôm not judging4.

Projects plugin

I set up a Project over my Notes - sources folder, with a Table view. I pull in a few metadata fields that make finding things easy - note filename, first author, title, article type, year. At present there‚Äôs no proper date handling so year just looks like a large number, but it still works for sort/filter. An extra nice thing about Projects is that it forces me to make sure my note metadata is actually correctly populated and formatted6.

Workflow example

I find a reference I want to collect, and maybe even read. Delightful! I rub my little hands together gleefully and prepare to add it to Zotero. If its a new reference it‚Äôll generally have a DOI, which is the best of all possible worlds. I click the ‚Äòmagic wand‚Äô icon in Zotero, paste in the DOI, and the reference appears in my collection.

If a full-text PDF is freely available and scrapeable, Zotero finds and downloads it for me. There‚Äôs a right-click > ‚ÄòFind available PDF‚Äô option that may need to be triggered. If that doesn‚Äôt work, I do my best to find that fulltext myself7, and use Zotfile‚Äôs context menu to add it as an attachment and automatically rename it and move it to ‚ú®The Hoard‚ú®. I do my little extra tidying steps, and that‚Äôs that.

For older references, I add in the details and attach, move and rename files manually.

Once I want to start reading and taking notes, I move to Obsidian and use ctrl+shift+o to open the Citations dialog box, search and select the reference I want, and click to create a new note. The note appears, ready with basic metadata, places to add text and tags, and working links back to a) the Zotero entry itself and b) the source document. I can open the source document in Obsidian by using ctrl+click on the PDF link, and can pin it in the app alongside my notes page, letting me read and write easily.

Why don‚Äôt I‚Ä¶?

  • Take notes in Zotero? I used to, but it has very limited capability for cross-linking between notes, so no real capacity for generating syntheses. I like to do things like build literature timelines, and Obsidian is simply a better place for this. Zotero‚Äôs notes attachments are handy for a few things like meta notes I don‚Äôt want to share (‚Äúthis paper sucks!‚Äù) or PDF TOC exports, but otherwise I don‚Äôt find them useful.

  • Use Zotero‚Äôs ‚ÄòAttach Stored Copy‚Äô functionality? Because then I can‚Äôt easily get at the document except by using Zotero, and often I just want to find a source quickly and e.g. email it to someone. Stored copies on Windows wind up in \\Users\%USERNAME%\Zotero\storage\%RANDOMFOLDER%, where %RANDOMFOLDER% is an alphanumeric internally associated with the reference entry, making them poorly discoverable.

  • Annotate PDFs and export? Basically I don‚Äôt remember anything I highlight, that‚Äôs just pretty colours. I have to rewrite things in my own words to take them in properly. I also work with a lot of old, bad PDF scans with crappy OCR or none at all, so it can be frustrating to even try. People who lean on a PDF-annotation workflow seem to never have to access anything older than 5-10 years, and that makes me equal parts jealous and suspicious.

Things I like

  • I‚Äôm actually using this, instead of setting it up and then ignoring it!
  • I‚Äôm actually using this, instead of setting it up and then endlessly trying to optimise it!
  • If I don‚Äôt use this for a few weeks, its not impossible to remember how everything works together.
  • Within notes I can add additional links to specific pages in the source pdf, using syntax like [[Darwin_1859_origin-Species-1e.pdf#page=112|Chapter 4 - Natural Selection]]. This is fabulous.
  • I can also do (as needed) a bunch of other stuff like add stored images, paste in screenshots, attach/link to other files and other notes, embed maps, and add other kinds of rich content like youtube videos to my notes.
  • I can link all this stuff to my ‚Äòdaily notes‚Äô diary setup, also in Obsidian, and do some productivity/focus tracking and task management - a topic for another time8.

Things I don‚Äôt like

  • OneDrive has to be running and logged in properly before interacting with the other software, otherwise things won‚Äôt sync correctly.
  • I have to be careful to update Obsidian, Zotero, and the necessary plugins on both my home and work machines. I also need to close Obsidian before moving from one physical device to another to prevent sync issues.
  • My use of the ‚ÄòShort Title‚Äô field in Zotero is decidedly off-label and may cause problems in the long term. I also have to populate it manually before starting a new note, so adding a new reference takes longer.
  • Re-addressing all the linked file attachments after moving ‚ú®The Hoard‚ú® seems to be a high-risk operation; somehow I didn‚Äôt quite get it right the first time I moved it inside my Vault and have been slowly repairing links in Zotero ever since üòï At least my naming convention makes things easy.
  • Obsidian‚Äôs table support sucks, because tables suck when implemented in any plaintext format. I need and love tables. I don‚Äôt see an immediate solution for this. I‚Äôd at least be happy if I could reliably copy+paste markdown table data between different apps, but that‚Äôs not a thing right now.
  • Zotero is annoying in some ways. Despite being definitely overall best in show for referencing:
    • it lacks specific reference types for some things that really should be handled better, like datasets, databases, and preprints
    • It only enables the DOI field on a few item types, but people will whack a DOI on anything these days. I don‚Äôt like having to store it in the ‚Äòextra‚Äô field!
    • The Windows GUI has a number of non-critical but highly irritating bugs, like laggy tabbing between fields.
    • Mass editing capability isn‚Äôt quite there yet. I‚Äôd like to do some clean-up and consistency work, but that‚Äôs not easy right now.
    • The help forum has that clueless-newb vs hostile-oldtimer vibe so common to software and programming support. Its pretty mild compared to a lot of places, but still tiresome.
  • The ‚Äòlinked files‚Äô approach is good for me but maybe not optimal for sharing with colleagues‚Ä¶but I can still potentially extend read access to ‚ú®The Hoard‚ú®over OneDrive and they can import the *.bib file as needed - there‚Äôs just no auto-sync. I haven‚Äôt tried this yet - there would certainly be some issues around integrating with their existing reference library that I don‚Äôt want to have to debug.
  • Exporting notes in a normie-friendly format is a problem. Most of my colleagues don‚Äôt use Obsidian or markdown. I‚Äôve said it before, but some kind of Quarto-based Obsidian-alike would be amazing, then I could throw html files or PDFs at them all day.

Anyway, that‚Äôs that on that.
